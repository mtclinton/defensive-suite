At least one detection path must not run on the potentially-compromised kernel. A live host with an eBPF rootkit lies to ps, ss, bpftool and every eBPF-based EDR — so bpfsentry adds early-boot baselining and offline memory forensics, and any divergence between the live bpftool view and the memory-dump prog_idr walk is itself proof of a sys_bpf-hooking implant. Trust anchors (AIDE DBs, baselines, BPF allowlists) live off-host, writable but not readable by the monitored machine.
